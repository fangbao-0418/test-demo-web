#### 元字符
```
  "^" : ^会匹配行或者字符串的起始位置，有时还会匹配整个文档的起始位置。
  "$" : $会匹配行货字符串的结尾
  "\b" : 不会消耗任何字符只匹配一个位置，常用语匹配单词边界 如 我想冲字符串中"This is RegExp" 匹配单独的单词 "is" 正则就要写成 "\bis\b" \b 不会匹配is 两边的字符，但它会识别is两边是否为单词的边界
  "\d" : 匹配数字
  "\w" : 匹配字母、数字、下划线
  "\s" : 匹配空格
  "." : 匹配除换行符以外任意字符 "\w"的加强版
  "[abc]" : 字符组 匹配包含括号内元素的字符
  以下几种反义
  "\W" ： 匹配任意不是字母、数字、下划线的字符
  "\B" : 匹配任意不是单词开头或结束的位置
  "\S" : 匹配任意不是空白符的字符
  "\D" : 匹配任意非数字的字符
  "[^abc]" : 匹配除了abc以外的任意字符
```

#### 懒惰限定符
```
  "*?" : 重复任意次，但尽量少重复
      如 "acbacb"  正则  "a.*?b" 只会取到第一个"acb" 原本可以全部取到但加了限定符后，只会匹配尽可能少的字符 ，而"acbacb"最少字符的结果就是"acb"
  "+?" : 重复一次或更多次，但尽量少重复  
      与上面一样，只是至少要重复1次
  "??" : 重复0次或1次，但尽量少重复
  "{n,m}?" : 重复n到m次，但尽量少重复
  "{n,}?" : 重复n次以上，但尽量少重复    
```

#### 捕捉组
```
  正则中捕捉分组其实就是一个括号内的内容 如"(\d)\d" 而"(\d)" 这就是一个捕捉分组，可以对捕捉分组进行 后向引用(如果后而有相同的内容则可以直接引用前面定义的捕捉组，以简化代表式) 如(\d)\d\1 这里的"\1"就是对"(\d)"的后向引用
  如 "zery zery" 正则 \b(\w+)\b\s\1\b 所以这里的"\1"所捕捉到的字符也是与(\w+)一样的"zery",为了让组名更有意义，组名可以自定义名字的
  "\b(?<name>\w+)\b\s\k<name>\b" 用"?<name>"既可以自定义组名了而要后想引用要记得写成"\k<name>";自定义组名后，捕捉组中匹配到的值就会保存到定义的组名里

  (exp) : 匹配exp,并捕捉文本到自动命名的组里
  (?<name>exp) : 匹配exp,并捕捉文本到name组里
  (?:exp) : 匹配exp,不捕获匹配的文本,也不给此分组命名
  (?=exp) : 匹配exp前边的位置
      如 "How are you doing" 正则"(?<txt>.+(?=ing))" 这里取ing前所有的字符，并定义了一个捕获分组名字为 "txt" 而"txt"这个组里的值为"How are you do";
  (?<=exp) : 匹配exp后边的位置
      如 "How are you doing" 正则"(?<txt>(?<=How).+)" 这里取"How"之后所有的字符，并定义了一个捕获分组名字为 "txt" 而"txt"这个组里的值为" are you doing";
  (?!exp) : 匹配后边跟的不是exp的位置
      如 "123abc" 正则 "\d{3}(?!\d)"匹配3位数字后非数字的结果
  (?<!exp) : 匹配前边不是exp的位置
      如 "abc123 " 正则 "(?<![0-9])123" 匹配"123"前面是非数字的结果也可写成"(?!<\d)123"
```
